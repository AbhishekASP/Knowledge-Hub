The six pillars of data quality—often called dimensions—are core criteria used to measure whether information is reliable and fit for its intended purpose. Evaluating data against these metrics prevents flawed decisions and operational inefficiencies.
    Accuracy: Data must correctly reflect real-world facts and events. It must be free from errors or misrepresentations.
    Completeness: All required records and necessary data elements must be present, with no missing values or blank fields.
    Consistency: Data values must remain uniform and coherent across different departments, datasets, and systems.
    Timeliness: Data must be fresh, up-to-date, and available exactly when it is needed for analysis or business processes.
    Validity: Information must strictly adhere to defined business rules, constraints, and syntax formats (e.g., proper date layouts or email structures).
    Uniqueness: There should be no duplicate pieces or redundant overlaps of data within a given dataset.
